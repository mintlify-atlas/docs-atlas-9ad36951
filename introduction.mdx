GNU Hyperbole (pronounced Ga-new Hi-per-bo-lee) is like Markdown for hypertext. It automatically recognizes dozens of common patterns in any buffer — regardless of mode — and activates them as hyperbuttons with a single key. Email addresses, URLs, grep output lines, git commits, programming backtraces, Org links, Info cross-references, and more all become instantly clickable. Load Hyperbole once and your text comes to life with no extra effort and no markup required.

Hyperbole is a single global minor mode with no dependencies outside standard Emacs libraries. It works with GNU Emacs 28 or above. It is designed and written by Bob Weiner and maintained by him and Mats Lidell.

## The six components

Hyperbole consists of six integrated parts. The whole is greater than the sum: the components work together to form a unified information management environment.

## What Hyperbole can do for you

### Personal information management

Overlapping link paths provide multiple views into an information space. A single key press activates any button regardless of its type — link, Lisp expression, or shell command — so navigation stays consistent everywhere. A search facility locates buttons in context for quick selection.

### Documentation browsing

Embed cross-references in any document format. Add point-and-click navigation to existing documentation without modifying its format. Link code and design documents, and jump to the definition of an identifier directly from its use in code or its mention in documentation.

### Brainstorming

Capture ideas and reorganize them rapidly with the Koutliner. Link to related ideas instead of copying and pasting, so your information stays in one authoritative place and links connect the concepts.

### Hypertextual email and news

Embed Hyperbole buttons in email messages. Recipients who also use Hyperbole can activate the buttons directly from their mail reader. Hyperlinks can include Emacs Lisp variables so they resolve correctly at different sites.

### Help and training systems

Create interactive tutorials with embedded buttons that demonstrate concepts hands-on, for example walking a reader through shell commands by letting them press a key to execute each step.

## How buttons work

There are three categories of Hyperbole buttons:

- **Explicit buttons** — manually created and embedded in a document, visible as `<(button label)>`. Created by dragging between windows or via the `C-h h e` menu.
- **Global buttons** — explicit or named implicit buttons stored in your personal button file (`C-h h b p`). Accessible by name from anywhere without the file being on screen.
- **Implicit buttons** — patterns that Hyperbole recognizes automatically in any buffer: URLs, file paths, grep output lines, git commit hashes, Info references, Org links, key sequences in braces, and many more. No creation step required.

Press `M-RET` (the Action Key) on any of these to activate it. Press `C-u M-RET` (or `C-h A`) first to preview what it will do before committing.

## Relationship to Org mode

Hyperbole and Org mode are complementary, not competing. Hyperbole works across **all** Emacs modes, while Org mode operates only within Org files. This means all Hyperbole functionality — including implicit button recognition — is available inside Org files as well. HyWiki, Hyperbole's wiki system, uses Org mode for its note files. Use both together.
