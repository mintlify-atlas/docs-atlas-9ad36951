This guide gets you from a fresh Hyperbole installation to a productive session as quickly as possible. You will learn the three keys you need for everything, then explore the built-in interactive demos.

<Note>
  This guide assumes Hyperbole is already installed and `hyperbole-mode` is active. If you see `Hypb` in your modeline, you are ready. If not, see the [Installation](/installation) guide first.
</Note>

## The three essential keys

Everything in Hyperbole flows from three key bindings. Learn these and you can navigate the entire system.

| Key | Name | What it does |
|---|---|---|
| `M-RET` | Action Key | Activates whatever is at point: a button, URL, file path, grep line, key sequence, and more |
| `C-u M-RET` | Assist Key | Shows help for what the Action Key would do at point, without activating it |
| `C-h h` | Hyperbole menu | Opens the main Hyperbole menu in the minibuffer for keyboard-driven navigation |

<Tip>
  You can also press `C-h A` (capital A) to preview what the Action Key will do at point. Use this any time you are unsure what `M-RET` will activate.
</Tip>

## Onboarding flow

<Steps>
  <Step title="Confirm Hyperbole is active">
    Press `C-h h`. You should see the Hyperbole menu appear in the minibuffer window at the bottom of your frame. It looks something like:

    ```
    Hyperbole: Act Butfile Doc Cust Find Gbut Hist Ibut Kotl Msg Rolo Screen Wish
    ```

    Type the first letter of any item to select it, or press `q` to quit. If the menu does not appear, make sure `(hyperbole-mode 1)` is in your init file and you have restarted Emacs.
  </Step>
  <Step title="Try the Action Key on implicit buttons">
    Open any buffer — a source file, a shell buffer, an email, anything. Move point over one of the following kinds of text and press `M-RET`:

    - A URL (e.g., `https://www.gnu.org/software/hyperbole/`)
    - A file path (e.g., `~/.emacs`)
    - A grep output line (e.g., `hbut.el:42: ...`)
    - A key sequence in braces (e.g., `{C-h h}`)
    - An email address

    Hyperbole recognizes each pattern and does the right thing automatically: open a browser, visit the file, jump to the source line, execute the key sequence, or compose mail.

    <Tip>
      If you are unsure what will happen, press `C-u M-RET` first. This runs the Assist Key and describes the action without performing it.
    </Tip>
  </Step>
  <Step title="Run the interactive DEMO">
    Press `C-h h d d` to open the Hyperbole DEMO file in Emacs. This is an interactive walkthrough of Hyperbole's button capabilities — the fastest way to build muscle memory for the Action Key.

    The DEMO file itself contains dozens of implicit buttons. Press `M-RET` on anything that looks interesting: quoted file paths, brace-delimited key sequences, angle-bracket action buttons, and table entries are all live hyperlinks inside the demo.

    When you are done, press `q` or close the buffer normally.
  </Step>
  <Step title="Try the Koutliner demo">
    Press `C-h h k e` to open an editable copy of Hyperbole's example Koutline file. The Koutliner is Hyperbole's multi-level, autonumbered outliner.

    Inside the Koutliner demo you can:

    - Read the embedded explanations of Koutliner commands
    - Edit cells and watch autonumbers update automatically
    - Move subtrees and observe the permanent cell IDs stay attached

    See the [Koutliner](/features/koutliner) page for a full command reference.
  </Step>
  <Step title="Try HyControl window management">
    Press `C-h h s w` to enter HyControl window control mode. HyControl gives you single-key commands to resize, move, and arrange your Emacs windows and frames.

    While in window control mode:

    - Arrow keys and other single keys adjust window sizes and positions
    - Press `t` to switch between window control and frame control
    - Press `q` to exit HyControl and return to normal editing

    You can also press `C-c \` at any time (if Hyperbole bound it during startup) to jump directly into HyControl window control.

    For frame control instead, press `C-h h s f`.
  </Step>
</Steps>

## Key binding reference

The most important Hyperbole key bindings at a glance:

| Key | Command | Description |
|---|---|---|
| `M-RET` | Action Key | Activate button or implicit button at point |
| `C-u M-RET` | Assist Key | Describe what Action Key would do at point |
| `C-h A` | `hkey-help` | Describe Action Key context (same as Assist Key) |
| `C-h h` | `hyperbole` | Open the Hyperbole main menu |
| `C-h h d d` | Demo | Launch the interactive DEMO |
| `C-h h d a` | About | Display the HY-ABOUT file |
| `C-h h d i` | Info | Browse the Hyperbole manual in Info format |
| `C-h h k e` | Koutliner example | Open the Koutliner demo/example file |
| `C-h h s w` | Screen → Windows | Enter HyControl window control mode |
| `C-h h s f` | Screen → Frames | Enter HyControl frame control mode |
| `C-h h b p` | Personal button file | Open your global button file for editing |
| `C-h h g a` | Global button activate | Activate a named global button by name |
| `C-h h e c` | Explicit button create | Create an explicit button at point |
| `C-h h i l` | Implicit link create | Create an implicit link button between two windows |
| `C-h h f` | Find menu | Open the Find/Web search menu |
| `C-c \` | HyControl (quick) | Jump to HyControl window control (if bound) |
| `C-c @` | Window grid | Create a grid of windows from recent buffers |

<Note>
  All Hyperbole menu commands are accessible from the keyboard via `C-h h` followed by the first letter of each submenu item. You can also invoke any command interactively with `M-x` using its full name.
</Note>
